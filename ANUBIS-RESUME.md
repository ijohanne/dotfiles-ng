## OVH Kimsufi deployment rules (DO NOT FORGET)

1. **Rescue mode is netboot** â€” every reboot returns to rescue until you change it in OVH panel
2. **Kimsufi has NO KVM/IPMI** â€” if you brick the boot, rescue mode is your only option
3. **Always use `--phases kexec,disko,install`** (no reboot) with nixos-anywhere so the user can change netboot to harddisk BEFORE rebooting
4. **kexec from rescue can trigger OVH "intervention"** which blocks netboot changes temporarily
5. **`boot.swraid.enable = true`** is required for mdadm â€” disko does NOT set this automatically
6. **Verify critical boot config with `nix eval`** before deploying to a server with no console access
7. **Each failed boot costs significant time** â€” get it right the first time
8. **The age key changes on every nixos-anywhere run** â€” always update `.sops.yaml` and `sops updatekeys` after install, before rebooting
